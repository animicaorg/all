# Indices · Canonical Links & Endpoints

This page centralizes public entry points for the **website**, **explorer**, **studio**, **installers**, and **seed lists**. Use these when writing docs, release notes, or support replies so we stay consistent across the project.

> **Note:** Hostnames below are examples/placeholders until mainnet GA. For local/test usage, prefer the **testnet** and **local** rows.

---

## 1) Primary Surfaces

| Surface    | Prod (mainnet)                  | Testnet (public devnet)                 | Local (developer)                       |
|------------|----------------------------------|-----------------------------------------|-----------------------------------------|
| **Website** | https://animica.org              | https://testnet.animica.org              | http://localhost:4321                    |
| **Explorer (Web)** | https://explorer.animica.org     | https://explorer.testnet.animica.org     | http://localhost:5173                    |
| **Studio (Web IDE)** | https://studio.animica.org       | https://studio.testnet.animica.org       | http://localhost:5174                    |
| **Studio Services API** | https://services.animica.org     | https://services.testnet.animica.org     | http://localhost:8080                    |
| **RPC (HTTP)** | https://rpc.animica.org            | https://rpc.testnet.animica.org          | http://localhost:8545                    |
| **RPC (WebSocket)** | wss://rpc.animica.org/ws          | wss://rpc.testnet.animica.org/ws         | ws://localhost:8546/ws                   |

### Deep Links (computed)
- **Open Studio**: `https://studio.animica.org/?rpc=<RPC_URL>&chainId=<ID>`
- **Open Explorer** (address): `https://explorer.animica.org/address/<bech32>`
- **Docs splash**: `https://animica.org/docs`

> In the website app, these are generated by `src/config/links.ts` using `PUBLIC_*` env vars. See `website/.env.example`.

## 3) Chain Metadata

Authoritative chain registry in-repo:
- **Spec registry**: `spec/chains.json`
- **Website bundle** (generated at build):  
  - `website/chains/index.json` (index)  
  - `website/chains/animica.mainnet.json`  
  - `website/chains/animica.testnet.json`  
  - `website/chains/animica.localnet.json`

API mirrors (served by the website):
- `GET /api/chainmeta.json` → merged set (see `website/src/pages/api/chainmeta.json.ts`)

## 5) Health & Status

- **Site health**: `GET https://animica.org/api/healthz` (or `/healthz` on services)
- **Chain status JSON**: `GET https://animica.org/api/status.json` → head height/TPS (configured via `PUBLIC_RPC_URL`)
- **Explorer WS**: `wss://rpc.animica.org/ws` → `newHeads`, `pendingTxs`

---

## 6) Updating This Index

- Submit PRs updating links here **and** the environment files:
  - `website/.env.example`
  - `installers/.env.example`
- For new networks:
  1. Add entry in `spec/chains.json`
  2. Mirror to `website/chains/*.json`
  3. Update `website/chains/index.json`
  4. Ensure seeds & RPC endpoints are live and monitored
